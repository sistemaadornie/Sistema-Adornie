Como reorganizar o modal "Dados do Pedido"?

Hoje tudo fica empilhado verticalmente (Ações → Pendências → Informações → Endereço → Itens → Pagamentos → Observações → Mídias). Aqui estão 3 direções para dar mais destaque ao essencial e reduzir a sensação de "lista infinita".

A — Resumo fixo + sub-abas

Um cabeçalho de resumo (cliente, status, total, pendências) sempre visível no topo do modal. Abaixo, as seções viram sub-abas internas (Informações / Itens / Pagamentos / Mídias) — você só vê uma de cada vez, sem rolagem longa.

B — Coluna lateral fixa + conteúdo rolável

Painel esquerdo fixo com identidade do pedido (cliente, status, total, pendências) e ações principais. À direita, o conteúdo detalhado (itens, pagamentos, observações, mídias) rola independente — o essencial nunca some de vista.

C — Cards colapsáveis (accordion)

Banner de resumo no topo. Cada seção (Informações, Itens, Pagamentos, Observações, Mídias) vira um card que expande/recolhe — o usuário abre só o que precisa, mantendo a visão geral compacta.

Minha recomendação

Eu pencho para a opção A (resumo fixo + sub-abas): resolve diretamente os 3 pontos que você apontou — destaca o essencial (cliente/status/total/pendências sempre visíveis), quebra o "empilhamento" em seções navegáveis, e abre espaço para repaginar o formulário de edição dentro da própria sub-aba "Informações" sem competir com itens/pagamentos. A opção B é boa em telas largas mas desperdiça espaço em modais (que já são limitados a ~920px). A opção C reduz a rolagem mas exige mais cliques para ver tudo de uma vez — pior para quem revisa o pedido inteiro.
