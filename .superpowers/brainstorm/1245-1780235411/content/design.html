2. O que muda no código

Antes (comportamento atual)
  • Página inteira rola (scroll global)
  • Detalhe usa position: sticky
  • Lista cresce infinitamente para baixo
  • Clicar item no fundo → detalhe no topo
Depois (novo comportamento)
  • Só as colunas rolam (scroll interno)
  • Detalhe usa height: 100%; overflow-y: auto
  • Layout ocupa altura do viewport fixo
  • Clicar qualquer item → detalhe sempre visível

3. Comportamento mobile (< 900px)

Em telas pequenas, as colunas ficam empilhadas. Ao clicar um item, a página rola automaticamente para mostrar o detalhe (scrollIntoView). Sem mudança visual no mobile.

Desktop (≥ 900px)
Painéis independentes lado a lado — sem mudança de posição ao selecionar
Mobile (< 900px)
Lista empilhada, ao clicar: scroll automático suave até o detalhe
Escopo da mudança
CSS de Pedidos.css e Clientes.css — nenhum outro arquivo de estilo global é alterado. O JSX das páginas recebe o ref para scroll no mobile.
